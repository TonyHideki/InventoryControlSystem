🧾 在庫管理システム（CLI版） 要件定義書（ドラフト）
1. システム概要

　本システムは、コンソール（CLI）上で動作する在庫管理ツールである。
　ユーザーは、商品の登録・更新・削除・検索・一覧表示などを行うことができる。
　商品データは、CSVファイルに保存し、次回起動時に読み込まれる。

2. 開発目的

　Javaの基礎文法（クラス設計、コレクション、ファイル入出力、例外処理）の学習

　実務を意識した小規模アプリケーション開発の経験を積む

　ポートフォリオとして公開できる成果物を作成する

3. 利用環境

　項目			内容

　OS			Windows / macOS / Linux
　実行環境		Java 17 以降（JDK 17以上）
　実行方法		コマンドライン上で java InventoryApp を実行
　データ保存		CSVファイル（例：inventory.csv）

4. 対象データ（商品情報）

　項目名		データ型	必須	説明

　商品ID		int		○	一意の識別子
　商品名		String		○	商品の名前
　在庫数		int		○	現在の在庫個数
　単価			int		○	商品1つあたりの価格（円）

5. 機能要件

　① 商品登録機能

	新しい商品を登録する

	ID重複チェックあり

	在庫数・単価は数値入力

　② 商品一覧表示機能

	登録されている全商品の情報を一覧で表示する

	表形式（整列表示）

　③ 商品検索機能

	IDまたは名前で検索

	該当商品があれば詳細表示

　④ 商品更新機能

	在庫数や単価を変更できる

	商品名の変更も可能

　⑤ 商品削除機能

	指定IDの商品を削除

　⑥ データ保存・読み込み機能

	起動時にCSVファイルを読み込み

	終了時にCSVファイルへ書き込み

　⑦ 終了機能

	プログラム終了時に保存を実行し、終了メッセージを表示

6. 非機能要件

　項目			内容
　操作性		数字入力でメニュー操作できるCLI
　保守性		クラスごとに責務を分離（単一責任原則）
　拡張性		GUI化やDB連携を視野に入れた設計
　データ永続性		CSV保存による永続化対応
　安全性		入力値検証（数値・重複チェックなど）を行う
